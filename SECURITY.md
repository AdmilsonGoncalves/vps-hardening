# Security Policy

The `vps-hardening` project takes infrastructure and server security extremely seriously. We appreciate the efforts of security researchers, systems engineers, and open-source contributors in helping keep our scripts and hardening recommendations robust and secure.

---

## Supported Versions

Only the current main/master baseline receives active security updates and auditing.

| Version / Branch | Supported          |
| ---------------- | ------------------ |
| `master` (latest)| :white_check_mark: |
| Older releases   | :x:                |

---

## Reporting a Vulnerability

If you discover a security vulnerability, flaw in our firewall/OpenSSH baseline, or potential lock-out hazard within our hardening scripts, **please do not open a public issue immediately**.

Instead, report the issue privately using **GitHub Private Vulnerability Reporting**:

1. Navigate to the **[Security tab](https://github.com/AdmilsonGoncalves/vps-hardening/security/advisories)** of this repository.
2. Click on **Report a vulnerability**.
3. Provide a clear description of the vulnerability along with steps to reproduce or script execution traces.

### Response Timeline
- **Acknowledgement**: Within 48 hours of your report.
- **Triage & Verification**: Within 5 business days.
- **Patch & Advisory**: Once verified, a security fix will be prepared and released promptly before any public disclosure.

We are committed to maintaining open communication throughout the remediation process and giving proper credit to researchers who report vulnerabilities responsibly.
